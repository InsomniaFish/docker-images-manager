# 使用演示

## 🎬 功能演示

### 1️⃣ 查看镜像列表

启动服务后，打开浏览器访问 http://localhost:8000

页面会自动显示所有 Docker 镜像，包括：
- 镜像名称（如 `nginx:latest`）
- 镜像 ID（短格式，如 `a1b2c3d4e5f6`）
- 镜像大小（如 `142.5 MB`）
- 创建时间（如 `2024-01-15 10:30:00`）
- 状态标识（🟢 使用中 / 🔴 未使用）
- 使用该镜像的容器数量

### 2️⃣ 选择镜像

#### 方式一：手动勾选
点击每行前面的复选框来选择或取消选择镜像

#### 方式二：使用快捷按钮
- **全选**：选择所有镜像
- **取消全选**：取消所有选择
- **选择未使用**：智能选择所有未使用的镜像（推荐）

### 3️⃣ 删除镜像

1. 勾选要删除的镜像
2. 点击"🗑️ 删除选中"按钮
3. 在确认对话框中查看将要删除的镜像列表
4. 如果有镜像正在使用，会显示警告提示
5. 如需强制删除，勾选"强制删除"选项
6. 点击"确认删除"按钮

### 4️⃣ 查看删除结果

删除完成后会显示：
- ✅ 成功删除的镜像数量
- ⚠️ 失败的镜像及原因
- 删除操作详情

页面会自动刷新，显示最新的镜像列表

### 5️⃣ 刷新列表

点击"🔄 刷新"按钮可以手动刷新镜像列表，无需刷新整个页面

## 📸 界面说明

### 顶部统计栏
显示四个统计指标：
- 总镜像数
- 正在使用
- 未使用
- 总大小

### 工具栏
左侧按钮：
- 全选
- 取消全选
- 选择未使用

右侧按钮：
- 🔄 刷新
- 🗑️ 删除选中

### 镜像列表表格

| 列 | 说明 |
|---|---|
| 复选框 | 选择要删除的镜像 |
| 镜像名称 | 显示 repository:tag |
| 镜像 ID | 短格式 ID（12 位） |
| 大小 | 自动格式化的大小 |
| 创建时间 | 镜像创建时间 |
| 状态 | 使用中/未使用标识 |
| 容器数 | 使用该镜像的容器数量 |

### 选中信息栏
当选择镜像后显示：
- 已选择数量
- 选中镜像的总大小

## 🎯 最佳实践

### 场景一：清理未使用的镜像
```
1. 点击"选择未使用"按钮
2. 确认选中的都是不需要的镜像
3. 点击"删除选中"
4. 确认删除
```

### 场景二：批量清理特定镜像
```
1. 点击"全选"
2. 手动取消需要保留的镜像
3. 点击"删除选中"
4. 确认删除
```

### 场景三：强制删除问题镜像
```
1. 选择要删除的镜像（包括正在使用的）
2. 点击"删除选中"
3. 在确认对话框中勾选"强制删除"
4. 确认删除（⚠️ 谨慎操作）
```

## ⚡ 快捷键

- **ESC**：关闭删除确认对话框

## 💡 提示

1. **状态标识说明**
   - 🟢 绿色"使用中"：镜像正在被容器使用，删除前请确认
   - 🔴 红色"未使用"：镜像可以安全删除

2. **删除保护**
   - 默认情况下，正在使用的镜像无法删除
   - 强制删除可能导致容器无法运行

3. **实时统计**
   - 选择镜像时会实时显示选中的总大小
   - 帮助评估删除操作的影响

4. **错误处理**
   - 删除失败会显示具体原因
   - 常见原因：镜像正在使用、权限不足等

## 🔧 常见问题

### Q: 为什么有些镜像显示为 `<none>:<none>`？
A: 这些是悬空镜像（dangling images），通常是构建新镜像后留下的旧层。可以安全删除。

### Q: 删除镜像会影响运行中的容器吗？
A: 默认情况下，正在使用的镜像无法删除。如果强制删除，会影响容器的运行。

### Q: 如何批量删除所有未使用的镜像？
A: 点击"选择未使用"按钮，然后删除即可。

### Q: 删除操作可逆吗？
A: 不可逆。删除后需要重新拉取镜像。
